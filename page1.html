Элементы и атрибуты XML

XML (eXtensible Markup Language) — это язык разметки, который используется для хранения и передачи структурированных данных.
В отличие от HTML, XML не задаёт визуальное оформление — его задача заключается в описании структуры данных. Основные строительные блоки XML — это элементы и атрибуты.

  1. Элементы (Tags)

    Элемент XML представляет собой основную единицу данных. Он состоит из открывающего тега, содержимого и закрывающего тега:

                                        
<name>Abdul</name>
    • <name> - открывающий тег, обозначает начало элемента
    • Abdul - содержимое элемента
    • </name> - открывающий тег, конец элемента

    Пустые элементы можно записывать в сокращённой форме:

                                        
<city />

    Это эквивалентно <city></city>

  2. Закрывающие теги

    Каждый открывающий тег должен иметь соответствующий закрывающий тег. Это обязательное правило XML.

    Примеры:

                                        
<age>30</age>
<country>Austria</country>
    • Закрывающий тег повторяет имя открывающего, но с косой чертой /.
    • Несоответствие имён тегов или отсутствие закрывающего тега приведёт к ошибке парсинга.
  3. Вложенность элементов

    Элементы могут содержать другие элементы, формируя иерархическую структуру:

                                        
<person>
    <name>Abdul</name>
    <age>30</age>
        <address>
            <city>Berlin</city>
            <country>Germany</country>
        <address>
</person>
    • Закрывающий тег повторяет имя открывающего, но с косой чертой /.
    • Несоответствие имён тегов или отсутствие закрывающего тега приведёт к ошибке парсинга.

    Правила вложенности:

    1. Каждый родительский элемент может содержать любое количество дочерних элементов.

    2. Дочерние элементы должны быть полностью вложены — закрывающий тег родителя идёт после всех закрывающих тегов детей.

    3. Элементы не должны пересекаться

    ❌ Неправильно:

                                                
<parent>
    <child1>
    <child2></child2>
<child1>
</parent>

    ✅ Правильно:

                                                
<parent>
    <child1></child1>
    <child2></child2>
</parent>
  4. Атрибуты элементов

    Атрибуты позволяют хранить дополнительную информацию о элементе. Атрибуты размещаются внутри открывающего тега и задаются парами имя="значение":

                                        
<person id="123" role="admin">
    <name>Abdul</name>
</person>
    • id="123" — атрибут с идентификатором элемента
    • role="admin" — атрибут с ролью
    • Значения атрибутов всегда заключаются в кавычки (" или ').
    • Атрибуты должны иметь уникальные имена внутри одного элемента.
  5. Комбинирование элементов и атрибутов

    Элементы и атрибуты можно использовать вместе для построения гибкой структуры данных:

                                        
<book id="001" genre="fantasy">
    <title>The Hobbit</title>
    <author>J.R.R Tolkien</author>
    <year>1937</year>
</book>

    Здесь:

    • <book> - родительский элемент с атрибутами id и genre
    • <title>, <author>, <year> - дочерние элементы с текстом

  6. Синтаксис XML: ключевые правила

    1. Корневой элемент: весь документ XML должен иметь один корневой элемент, который охватывает всё содержимое.

    2. Чувствительность к регистру: <Name> и <name> — это разные элементы.

    3. Закрытие всех тегов: каждый открывающий тег должен иметь закрывающий.

    4. Корректная вложенность: элементы не должны пересекаться.

    5. Использование атрибутов: уникальные имена внутри одного элемента, значения в кавычках.

    6. Прозрачность пробелов: XML игнорирует лишние пробелы между тегами, но сохраняет их внутри текста.
  7. Пример корректного готового XML-документа 
                                        
<library>
    <book id="001" genre="fantasy">
        <title>The Hobbit</title>
        <author>J.R.R Tolkien</author>
        <year>1937</year>
    </book>
    <book id="002" genre="sci-fi">
        <title>Dune</title>
        <author>Frank Herbert</author>
        <year>1965</year>
    </book>
</library>
    • <library> - корневой элемент
    • <book> - элементы с атрибутами
    • <title>, <author>, <year> - дочерние элементы с текстом
